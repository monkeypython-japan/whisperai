# 0004. 英語への翻訳は言語別専用モデルを優先し、多言語モデルにフォールバックする

## 状況

英語への翻訳には当初 `Helsinki-NLP/opus-mt-mul-en`（多言語→英語の単一モデル）のみを使用していた。多言語対応の代償として、言語によっては専用モデルより翻訳品質が劣る。

## 決定

翻訳元言語ごとに `Helsinki-NLP/opus-mt-<言語>-en` の専用モデルが存在する場合はそれを優先して使用し、取得できない場合（モデルが存在しない・ダウンロード失敗）は `opus-mt-mul-en` にフォールバックする。対象言語は `config.toml` の `marian_source_langs` で管理する。

日本語など英語以外への翻訳は、専用モデルの組み合わせ数が非現実的に多くなるため、`facebook/nllb-200-distilled-600M`（NLLB-200）を単一モデルとして使用する。

## 理由

- 主要言語では専用モデルの方が翻訳品質が高い
- 一方で全言語ペア分の専用モデルを用意するのは非現実的なため、フォールバック方式で品質と網羅性を両立する
- NLLB-200 は多言語間翻訳を単一モデルでカバーでき、英語以外への翻訳先を増やす際の追加実装コストが低い

## 影響

- 初回実行時に、動画に含まれる言語の組み合わせに応じて追加のモデルダウンロードが発生し得る
- モデル名・対象言語リストは `config.toml` で調整可能（[[../../whisperai-spec.md|whisperai-spec.md]] の「設定ファイル」参照）
