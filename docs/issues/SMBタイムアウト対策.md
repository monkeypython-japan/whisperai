# ネットワークドライブ（SMB）への SRT 書き出しがタイムアウトすることがある

## 症状

処理対象の動画がネットワークドライブ（SMB マウント）上にある場合、SRT ファイルの書き出し（`open(path, "w")` での直接書き込み）がタイムアウト・失敗することがあった。

## 原因

SMB 越しのファイル書き込みは、ローカルディスクに比べてレイテンシや切断リスクが高い。

## 回避策

ローカルの一時ファイルに書き出してから `shutil.copy2` で最終出力先へコピーする方式に変更した（`output.py` の `write_srt`）。コピー失敗時は一時ファイルのパスと手動コピー用の `cp` コマンドを案内する。
