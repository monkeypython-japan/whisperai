# 音声認識(字幕トラック無し)で、実際は英語音声なのにロシア語等と誤検出されることがある

## 症状

英語音声のみの動画（`Thunderbirds.2004.1080p.BluRay.x264-[YTS.LT].mp4`、字幕トラック無し）を
言語コード省略で音声認識したところ、出力が `....ru.srt`（ロシア語）になった。中身を見ると
本編の台詞自体は正しく英語で書き起こされていたが、1ブロック目（00:00:17-00:00:19、
本編の台詞が始まる前の無音/導入部分）に次のハルシネーションが混入していた:

```
Субтитры создавал DimaTorzok
```

（ロシア語で「字幕作成: DimaTorzok」の意。Whisper系モデルで広く報告されている既知の
ハルシネーション定型句のひとつ）。`mlx_whisper.transcribe()` は言語コード未指定時、
先頭30秒のメル・スペクトログラムから言語を自動検出するが、この短いハルシネーション
片がロシア語として検出されたことで、動画全体の判定言語が `ru` になってしまった
（本編の書き起こし自体は最後まで英語のまま行われていたにも関わらず）。

## 原因

`transcribe()`（`transcribe.py`）が `mlx_whisper.transcribe()` を呼ぶ際、`language` を
指定しておらず常に自動検出に任せていた。`whisperai.py` の音声認識分岐
（`process_video()` の字幕なし/`--transcribe` 分岐）でも、位置引数として渡された
言語コード（`lang_code`）を一切参照しておらず、出力ファイル名も検出言語
（`detected_lang`）にのみ依存していた。

## 対処

- `transcribe()` に `language` 引数を追加し、`mlx_whisper.transcribe(..., language=language)`
  へ渡すようにした（`None` なら従来通り自動検出）。
- `whisperai.py` の音声認識分岐で、指定済みの `lang_code` を `transcribe()` に渡し、
  出力ファイル名も `lang_code or detected_lang` を使うようにした。これにより、
  音声言語が判明している場合は `whisperai.py "movie.mp4" en --transcribe` のように
  言語コードを明示すれば、無音区間のハルシネーションによる誤検出の影響を受けない。
- 呼び出し側の `subtitle-translation` は英語音声を前提とするパイプラインのため、
  `whisper_transcribe()`（`_lib.sh`）から常に `en` を明示指定するよう変更した
  （詳細は subtitle-translation リポジトリの ADR-0017 追記を参照）。

## 備考

言語コード未指定（汎用の自動検出モード）自体は既存動作のまま変更していない。
ハルシネーション自体（無音区間からの空耳クレジット行の生成）は
`no_speech_threshold` / `hallucination_silence_threshold` / `filter.py` の
クレジットパターンフィルターである程度抑制されるが、今回のロシア語フレーズは
`filter.py` の `credit_patterns`（英語パターンのみ）に一致せず素通りしていた。
本文中に紛れ込む単発の空耳行そのものを機械的に0にする対策ではない点は留意。
